A range is a min–max slider: two native `<input type="range">` overlaid on one
track. Each carries `.re-slider`, so the thumbs look exactly like a single
slider. With **zero JS** both inputs work, are keyboard-operable, and submit
their own `name=value`. `enhanceRange` keeps the thumbs from crossing, draws the
fill between them, and routes a track click to the nearer thumb.

```js
import { enhanceRange } from "@relements/core/behaviors/range";
enhanceRange(document);
```

## Notes

- Crossing is prevented by clamping the moved input's **value** — both inputs
  keep their authored `min`/`max` (the full range), so each thumb maps to the
  same track and `aria-value*` stays honest.
- `data-re-range-gap` sets a minimum distance between thumbs (default = `step`).
- When the two thumbs fully coincide, separate them with the keyboard (Tab to a
  thumb, then arrow) — the focused thumb sits on top.
- Native range thumb mirroring under `dir="rtl"` is engine-dependent; verify the
  fill alignment if you ship RTL.

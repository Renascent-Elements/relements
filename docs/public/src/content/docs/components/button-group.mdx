A button group joins related `.re-button`s into one control: the inner seams
collapse to a single hairline and only the group's outer corners stay rounded.
It's CSS-only — wrap the buttons in `.re-button-group` with `role="group"` and
an accessible name.

Members must be **direct** `.re-button` children and share a **uniform
variant**. Toggle members use native `aria-pressed`; disabled members use
`<button disabled>` or, for links, `<a class="re-button" aria-disabled="true">`.

## Vertical

Add `data-orientation="vertical"` to stack the members; the collapse and corner
rounding move to the block axis and `align-items: stretch` equalizes the widths.

## Link members

Any `.re-button` element works — including `<a>`. A disabled link uses
`aria-disabled="true"` (it stays keyboard-focusable).

## Notes

- For a band of unrelated controls with arrow-key roving, use the
  [Toolbar](/relements/components/toolbar/) instead.
- For a control with a **persistent selection** that submits with the form, use
  the [segmented control](/relements/components/segmented/) (one-of-N) or
  [toggle group](/relements/components/toggle-group/) (many-of-N). A button
  group is for stateless actions — its toggle members carry JS-only
  `aria-pressed`, not a submitted form value.
- The group sets no `overflow`, so a focused member's ring is never clipped; the
  focused member lifts above its neighbors via `z-index`.
- Everything is logical-property based, so it mirrors correctly in RTL.

## Accessibility

- **Keyboard** — entirely native and per-member; the group adds no roving
  tabindex or arrow-key navigation. <kbd>Tab</kbd>/<kbd>Shift+Tab</kbd> moves
  through each member in order; <kbd>Enter</kbd>/<kbd>Space</kbd> activates a
  `<button>` member and <kbd>Enter</kbd> follows an `<a>` member. (If you want
  arrow-key roving across a band of controls, reach for the
  [Toolbar](/relements/components/toolbar/) instead.)
- **Focus** — each member shows the standard visible `:focus-visible` ring
  (`--re-shadow-focus`); the group sets no `overflow` and lifts the focused
  member via `z-index`, so the ring is never clipped by a neighbor (see Notes
  above).
- **Semantics** — the container is `role="group"` with an accessible name
  (`aria-label`), so assistive tech announces the members as one labelled
  cluster. Members keep their native roles (`button`, `link`). Toggle members
  carry native `aria-pressed`, announced as a pressed/not-pressed state.
  Disabled members use `<button disabled>` (removed from the tab order) or, for
  links, `aria-disabled="true"` (announced disabled but still focusable).
- **Notes** — the join is purely visual: members stay independent focus stops,
  so the joined look never changes how the group is operated. See the
  [accessibility guide](/relements/guides/accessibility/) for the project-wide
  baseline.

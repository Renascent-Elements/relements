Relements ships two opt-in aids for authoring: **editor IntelliSense** (autocomplete
and hovers for the `<re-*>` tags, the `data-*` attributes, and the `--re-*` tokens),
and **TypeScript types** for the custom elements. Both are optional вҖ” nothing about
the library requires them.

## IntelliSense (autocomplete & hovers)

The package ships editor metadata in **two formats**, so the completions work beyond
VS Code:

- **[VS Code custom-data](https://code.visualstudio.com/api/extension-guides/custom-data-extension)**
  (`html.custom-data.json` + `css.custom-data.json`) вҖ” read by VS Code and by any editor
  that runs the same HTML/CSS language servers over LSP (Neovim, Helix, Zed, Sublime Text,
  Emacs).
- **[web-types](https://github.com/JetBrains/web-types)** (`web-types.json`) вҖ” read by
  JetBrains IDEs (WebStorm, IntelliJ, вҖҰ), **auto-discovered** from `node_modules` with no
  configuration.

Either way you get, in HTML and framework templates:

- the six `<re-tabs>` / `<re-menu>` / `<re-popover>` / `<re-toast>` / `<re-file-picker>` / `<re-tags-input>` tags,
  with a hover description and a docs link;
- the styling attributes вҖ” `data-variant`, `data-tone`, `data-size`, `data-placement`, вҖҰ
  (their allowed values, e.g. `data-tone=` вҶ’ `info` В· `success` В· `warning` В· `danger` В·
  `neutral` В· `hint`, are offered in VS Code and LSP editors);
- the declarative `data-re-*` behavior hooks (`data-re-dialog-trigger`, `data-re-dismiss`, вҖҰ);

and in CSS, **every `--re-*` token** inside `var(вҖҰ)`, with its default (and dark) value on
hover:

```css
.my-card {
  padding: var(--re-space-4); /* hover shows: `1rem` */
  color: var(--re-color-text);
}
```

### VS Code

Point your workspace at the two custom-data files in `.vscode/settings.json`:

```json
{
  "html.customData": ["./node_modules/@relements/core/html.custom-data.json"],
  "css.customData": ["./node_modules/@relements/core/css.custom-data.json"]
}
```

### JetBrains (WebStorm, IntelliJ)

Nothing to configure вҖ” JetBrains reads the package's `web-types` field automatically after
`npm install`. The `<re-*>` tags, `data-*` attributes, and `--re-*` tokens complete out of
the box.

### Other editors (Neovim, Helix, Zed, Sublime, Emacs)

These run the same language servers (`vscode-langservers-extracted`), which load the HTML
custom-data from the server's `dataPaths` init option. For example, with Neovim's built-in
LSP:

```lua
vim.lsp.config("html", {
  init_options = {
    provideFormatter = true,
    dataPaths = { vim.fn.getcwd() .. "/node_modules/@relements/core/html.custom-data.json" },
  },
})
```

The files are editor-agnostic JSON at `@relements/core/html.custom-data.json` and
`@relements/core/css.custom-data.json` вҖ” see your LSP client's docs for how it forwards
custom data (CSS custom-data is delivered via an LSP notification that some clients wire
up for you).

## TypeScript: typing the custom elements

Each `<re-*>` element ships a typed class (`ReTabsElement`, `ReMenuElement`,
`RePopoverElement`, `ReToastElement`) from its subpath. Two small declarations wire
those types into the DOM and into your framework's templates. Drop them in a
`relements.d.ts` that your `tsconfig.json` includes.

### Typed DOM lookups (every framework)

Augment `HTMLElementTagNameMap` so `document.querySelector` / `createElement` return
the right element type everywhere:

```ts
// relements.d.ts
import type { ReTabsElement } from "@relements/core/elements/re-tabs";
import type { ReMenuElement } from "@relements/core/elements/re-menu";
import type { RePopoverElement } from "@relements/core/elements/re-popover";
import type { ReToastElement } from "@relements/core/elements/re-toast";

declare global {
  interface HTMLElementTagNameMap {
    "re-tabs": ReTabsElement;
    "re-menu": ReMenuElement;
    "re-popover": RePopoverElement;
    "re-toast": ReToastElement;
  }
}

export {};
```

```ts
const tabs = document.querySelector("re-tabs"); // ReTabsElement | null
tabs?.value; // string | null вҖ” typed
```

### Typed templates (per framework)

So the `<re-*>` tags type-check inside your framework's templates, add the matching
augmentation. Each uses the element class from `@relements/core/elements/*`.

```ts
// relements.d.ts вҖ” React 19 (jsx: "react-jsx")
import type { DetailedHTMLProps, HTMLAttributes } from "react";

declare module "react/jsx-runtime" {

    }
  }

```

For `jsx: "react"` or `"preserve"`, augment `"react"` instead of `"react/jsx-runtime"`.

  </TabItem>
  <TabItem label="Vue">

Tell the Vue compiler the `re-*` tags are custom elements (runtime), then register the
types:

```ts
// vite.config.ts
vue({
  template: { compilerOptions: { isCustomElement: (tag) => tag.startsWith("re-") } },
});
```

```ts
// relements.d.ts
import type { HTMLAttributes } from "vue";
import type { ReTabsElement } from "@relements/core/elements/re-tabs";

declare module "vue" {
  interface GlobalComponents {
    "re-tabs": new () => ReTabsElement & {
      $props: HTMLAttributes & Partial<ReTabsElement>;
    };
  }
}
```

See Vue's [typing custom elements](https://vuejs.org/guide/extras/web-components.html#typing-custom-elements)
for the full event-typed helper.

  <TabItem label="Angular">

Angular doesn't type unknown tags, but it stops erroring on them once you add
`CUSTOM_ELEMENTS_SCHEMA` to the component/module that uses a `<re-*>` tag:

```ts
import { CUSTOM_ELEMENTS_SCHEMA, Component } from "@angular/core";

@Component({
  /* вҖҰ */
  schemas: [CUSTOM_ELEMENTS_SCHEMA],
})
export class MyComponent {}
```

The `HTMLElementTagNameMap` augmentation above still types any `ElementRef` /
`document.querySelector` lookups of the element.
